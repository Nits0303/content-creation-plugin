# SEO Skill

## Purpose

This skill performs SEO auditing and optimisation checks on content that has already been generated by the pipeline. It does not generate content. It does not restart intake. It does not modify CONTENT_BRIEF. It operates as a post-generation analysis layer triggered by the `/seo-check` command.

---

## Pipeline Position

```
/create → intake → router → generation skill → humanise → [DELIVERED CONTENT]
                                                                    ↓
                                                             /seo-check
                                                                    ↓
                                                             seo/SKILL.md
```

The SEO skill is not part of the main generation pipeline. It runs after content has been delivered to the user.

---

## Command Trigger

Triggered by: `/seo-check`

Example invocations:
- `/seo-check`
- "check SEO for this article"
- "optimise this page for SEO"
- "improve keyword placement"

---

## Inputs

The SEO skill receives:

1. `CONTENT_BRIEF` — read-only. Never modified.
2. The generated content — the full delivered output.
3. Optional user instruction — specific SEO focus if provided.

Fields read from `CONTENT_BRIEF`:

| Field | Use |
|---|---|
| `keyword` | Primary keyword against which all placement checks run |
| `format` | Determines which SEO rule set applies |
| `sub_type` | Determines subtype-level skip rules and schema selection |
| `sector` | Triggers Local SEO Conditional Check |
| `content_points` | Reference for LSI keyword alignment |
| `length_tier` | Context for internal link placeholder count |

---

## Format-Specific Behaviour

SEO check behaviour is determined by `CONTENT_BRIEF.format` before any individual check runs.

### Social — Full Exemption

When `format` is `social`: return the following notification and stop. Do not run any checks.

> SEO checks do not apply to social media content. No audit performed.

### Blog — Full Checks

When `format` is `blog`: all six SEO checks apply. Local SEO Conditional fires when sector qualifies.

Phase 4 skip rules by blog subtype:

- **Phase 4 SKIPPED for:** Opinion, News & Updates, Interview, Guest Post
- **Phase 4 FIRES for:** How-to, Listicle, Beginner's Guide, Ultimate Guide, SEO-Focused Article, Case Study, Product Review, Comparison Post

When auditing a blog subtype where Phase 4 is skipped (Opinion, News & Updates, Interview, Guest Post), keyword placement checks and LSI keyword checks must not run, even if a keyword is present in the brief. These are editorial content types and are not keyword-driven. For these subtypes: skip Check 1 and Check 2, and continue with Check 3 (meta title), Check 4 (meta description), Check 5 (schema), and Check 6 (internal links). Mark Check 1 and Check 2 as Not Applicable in the audit summary — do not flag them as failures.

### Webpage — Subtype-Dependent

When `format` is `webpage`: apply the subtype rules below.

**Keyword checks FIRE for:**

| Subtype | Qualifier |
|---|---|
| Landing Page | Always |
| Sales Page | Determined by `CONTENT_BRIEF.keyword`: if `keyword` is empty, treat the page as non-organic by default and skip keyword checks. If `keyword` is present, treat the page as organic-targeted and run keyword checks. |
| Product Page | Always |
| Services Page | Always |
| FAQ Page | Always |
| Case Study Page | Always |
| Industry Page | Always |
| Portfolio Page | Only when `CONTENT_BRIEF.sector` is `marketing-agency`. For all other sectors, treat as non-SEO-targeted and skip keyword checks. |
| Contact Page | Local businesses only (see Local SEO Conditional). For all other sectors, keyword checks are skipped. |

**Keyword checks SKIPPED for:**

| Subtype | How handled |
|---|---|
| Homepage | Brand terms applied silently. No keyword check. Meta title: Brand + core category. Meta description: value proposition summary. |
| Pricing Page | Brand + pricing terms applied silently. No keyword check. |
| About Page | Brand terms. No keyword check. |
| Testimonials Page | Brand + review terms. No keyword check. |
| 404 Page | Not an SEO target. Skip entirely. |
| Thank You Page | Not indexed. Skip entirely. |

For skipped subtypes: return a brief note stating which page type was detected and why keyword checks do not apply. Do not flag skipped checks as failures.

## SEO Checks

Six checks run on all qualifying content. Checks are run in order. Each check produces a pass, a flag with suggested correction, or a not-applicable notation.

### Check 1 — Keyword Placement

Verify the primary keyword (`CONTENT_BRIEF.keyword`) appears in:

- H1
- First 100 words of body content
- At least two H2 headings
- Meta title
- Meta description

For each missing placement: flag the specific location and provide a targeted correction. Do not rewrite surrounding content — adjust the specific element only.

### Check 2 — LSI Keywords

Five semantically related keywords must be present in the content. LSI keywords must be derived from the keyword cluster associated with `CONTENT_BRIEF.keyword` — they are not generated synthetically from the topic alone.

If `CONTENT_BRIEF.keyword` is absent: skip this check entirely. Do not generate synthetic keyword suggestions. Mark as Not Applicable in the audit summary.

If `keyword` is present, verify:
- Five LSI keywords are identifiable in the content
- Each appears naturally — not forced or clustered
- No single LSI keyword appears with unnatural density

If LSI keywords are absent or insufficient: generate the five LSI keywords and suggest placement locations. Placement suggestions must identify a specific sentence or paragraph, not a vague section reference.

Keyword stuffing is never an acceptable correction. If natural placement is not possible for a keyword, drop it from the suggestion.

### Check 3 — Meta Title

Meta title must follow the formula:

`Primary Keyword + Benefit/Angle + Brand`

Constraints:
- Maximum 60 characters
- Primary keyword must be present
- Brand name may be abbreviated or shortened to fit the character limit

If meta title is missing: generate one conforming title.
If meta title is present but fails one or more constraints: output the original and a corrected version side by side.

### Check 4 — Meta Description

Meta description must follow the formula:

`Action verb + keyword + value proposition`

Constraints:
- Maximum 155 characters
- Primary keyword must be present
- Must open with an action verb

If meta description is missing: generate one conforming description.
If present but non-conforming: output original and corrected version side by side.

### Check 5 — Schema Markup

Determine the appropriate schema type for the content based on `sub_type`. Output schema recommendation as a note at the end of the audit, not embedded in the content.

Schema type mapping:

| Sub-type | Schema |
|---|---|
| How-to / Tutorial | HowTo |
| FAQ Page | FAQPage |
| Product Page, Product Review | Product |
| Case Study, Case Study Page | Article |
| Blog (general) | Article |
| News & Updates | NewsArticle |
| Comparison Post | Article |
| Services Page, Landing Page | Service |
| Local sectors (real-estate, healthcare, ca_legal-finance, hospitality-travel, local-services) | LocalBusiness (in addition to primary schema) |

If the generated content already contains schema notes: verify they match the correct type. Flag mismatches.

### Check 6 — Internal Link Placeholders

Minimum two internal link placeholders are required for all blog posts and SEO-targeted web pages.

This check is **Not Applicable** for the following web page subtypes. Mark as Not Applicable in the audit summary — do not flag as a failure:

- Homepage
- Pricing Page
- About Page
- Testimonials Page
- 404 Page
- Thank You Page

Format:
```
[Internal link: topic]
```

Verify:
- At least two placeholders are present
- Each placeholder references a specific topic, not a vague descriptor
- Placeholders are placed within the body, not only at the end

If fewer than two placeholders exist: identify the two most logical locations in the content and output suggested placements. Do not add them directly — present as recommendations.

## Local SEO Conditional Check

Fires automatically when `CONTENT_BRIEF.sector` is one of:

- `real-estate`
- `healthcare`
- `ca_legal-finance`
- `hospitality-travel`
- `education`
- `local-services`

This check runs in addition to the six standard checks, not instead of them.

### What to verify when Local SEO fires:

**Location signals:** At least one specific location reference (city, suburb, service area, or region) must appear in the content. Flag if absent.

**Local keyword variations:** The content should include at least one local keyword variant — the primary keyword combined with a location modifier (e.g. "solicitor in Mumbai", "dental clinic Ahmedabad"). Flag if absent and suggest a placement.

**LocalBusiness schema:** Recommend LocalBusiness schema in addition to the primary schema type (see Check 5). This recommendation is additive — it does not replace the content-type schema.

**Contact Page specific behaviour:** For Contact Page in local sectors, keyword checks fire as a standard check (not skipped). Verify location signals and local keyword variants are present on the page.

If the sector qualifies but no location information is present in either the brief or the content: flag this as a gap. Do not invent location data.

---

## Hallucination Safeguards

The SEO skill operates under F.0.3 rules. SEO corrections are structural only.

The SEO skill must never introduce:
- Fabricated statistics or data points
- Fabricated quotes or attributed claims
- Research findings not present in the original content
- Synthetic authority signals ("studies show", "experts agree", "according to recent research") without a traceable current-session source
- Any factual claim not already present in the delivered content or the CONTENT_BRIEF

If a suggested correction would require adding a factual claim: flag the gap and instruct the user to supply the content. Do not generate the claim.

---

## Output Behaviour

The SEO skill returns an audit report. It does not rewrite the article.

### Output Structure

**1. SEO Audit Summary**
One-line status per check: Pass / Flag / Not Applicable

**2. Flagged Items**
For each flagged check: what is missing or incorrect, and the specific recommended correction. Corrections are targeted — they address only the failing element, not surrounding content.

**3. Meta Title**
If present and passing: confirm.
If missing or failing: output corrected version.

**4. Meta Description**
If present and passing: confirm.
If missing or failing: output corrected version.

**5. Schema Recommendation**
Primary schema type plus LocalBusiness addition if Local SEO Conditional fired.

**6. Local SEO Notes**
Present only when Local SEO Conditional fired. Lists location gaps and local keyword variant suggestions.

The user applies suggestions. The SEO skill does not apply them automatically.

---

## Prohibited Behaviour

The SEO skill must never:

- Restart intake
- Modify `CONTENT_BRIEF` in any field
- Generate new content or replace existing content
- Change subtype structure
- Alter CTA wording or CTAs
- Fabricate statistics, quotes, research findings, or authority claims
- Run web research unrelated to SEO verification
- Add factual claims not present in the delivered content or brief
- Flag skipped subtypes as SEO failures
- Apply Local SEO checks to sectors not in the qualifying list
- Run any check on social media content
