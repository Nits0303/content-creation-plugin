# /repurpose

---

## Command Purpose

`/repurpose` is the entry point for converting previously generated content into a different format. It signals that the user wants to produce a new content output from an existing one, in a different format. The command triggers the repurpose skill. It contains no repurposing logic.

---

## When This Command Is Used

Use `/repurpose` after content has been generated by `/create`, `/create-blog`, `/create-page`, or `/create-social`. It is used when the user wants to convert the existing output into a different content format rather than refine the current one.

## Repurpose Behaviour

`/repurpose` creates a new `CONTENT_BRIEF` for the target format. The new brief is not built from scratch â€” it inherits selected fields from the original brief and re-determines the rest.

**Fields inherited from the original brief:**

- `brand_name`
- `brand_description`
- `target_audience`
- `tone`
- `sector`
- `topic`
- `angle`

**Fields that must be changed or re-determined for the new format:**

- `format`
- `sub_type`
- `length_tier`
- `output_format`

**Fields that must not be inherited:**

- `keyword`
- `content_points`
- `cta_type`

These fields are format-specific. They are re-determined inside `repurpose/SKILL.md` based on the target format and subtype.

## Pipeline Activation

Invoking `/repurpose` starts the following pipeline:

```
/repurpose â†’ repurpose/SKILL.md â†’ router/SKILL.md â†’ [target generation skill] â†’ humanise/SKILL.md
```

The generation skill activated is determined by the target `CONTENT_BRIEF.format` constructed in `repurpose/SKILL.md`:

```
format = blog    â†’ blog/SKILL.md
format = webpage â†’ webpage/SKILL.md
format = social  â†’ social/SKILL.md
```

Unlike `/refine`, the router is re-entered. It reads the new `CONTENT_BRIEF.format` and activates the appropriate generation skill for the target format.

---

## What Happens After The Command Runs

1. `repurpose/SKILL.md` opens and loads the original `CONTENT_BRIEF`.
2. The user's target format is identified. If not stated explicitly, `repurpose/SKILL.md` asks.
3. A new `CONTENT_BRIEF` is constructed: inherited fields are carried over, format-specific fields are re-determined for the target format, and excluded fields are not carried over.
4. `router/SKILL.md` reads the new `CONTENT_BRIEF.format` and activates the correct generation skill.
5. The target generation skill (`blog/SKILL.md`, `webpage/SKILL.md`, or `social/SKILL.md`) generates the content using the new brief.
6. `humanise/SKILL.md` runs the humanise pass on the generated output.
7. The repurposed content is returned to the user.

---

## Output Behaviour

- Output format and subtype reflect the target format, not the original.
- Output structure, length, and SEO behaviour follow the rules of the target generation skill and its templates.
- Inherited fields (brand, audience, tone, sector, topic, angle) carry through into the new output without re-evaluation.
- Format-specific fields (keyword, content points, CTA type) are determined fresh for the target format and will differ from the original.
- Sector overlay and brand voice behaviour apply to the new generation using the inherited `sector` and `brand_description` values.

---

## Restrictions

- This command file contains no repurposing logic, brief construction logic, inheritance logic, generation logic, template logic, SEO logic, humanise logic, or sector overlay logic.
- All logic lives in the skill files activated by this pipeline.
- Do not modify this command file to add repurposing behaviour.
- Do not use this command to modify content within the same format. Use `/refine` for same-format adjustments.
- Do not use this command to generate new content from scratch. Use `/create`, `/create-blog`, `/create-page`, or `/create-social` for new generation.
